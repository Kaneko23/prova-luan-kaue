TDD:


    Em projetos é muito comum a utilização de testes para uma nova implementação de funcionalidades, sendo muito utilizada o TDD (desenvolvimento orientado a testes), podendo
ser utilizado quando é necessário fazer testes em um projeto, acontecendo antes ou durante o seu desenvolvimento. A maior parte das linguagens de programação já possuem
bibliotecas para utilização do TDD.


Importância de desenvolver um protótipo:


    Antes de um projeto ter sua finalização é necessário passar pela parte do protótipo do projeto, onde vai ser criado a primeira ideia de como irá ficar o projeto no final
e irá ter os ajustes de ideais, isso com o objetivo de aumentar a chance de sucesso do projeto.


Manifesto Ágil:


    O manifesto ágil foi criado para uma melhor forma de desenvolver um software, de uma maneira mais rápida e clara de ideias. O primeiro valor que é citado explica
 que se deve ter mais valores aos time que está programando do que nas ferramentas, melhor ter pessoas felizes e engajadas do que slids lindos que na verdade não representa
 a equipe que o fez.
    O segundo valor que é apresentado traz uma ideia onde um programa funcionando vale mais do que inúmeras linhas de documentos. O documento de um projeto é muito importante,
 Porém os clientes pagam para ter um resultado, portanto às vezes apresentar algo funcional vale mais do que apenas documentações na visão do cliente.
    Para que um projeto se desenvolva perfeitamente, deve-se ter uma comunicação e uma coesão de ideias entre a equipe de desenvolvimento e o cliente, é isso que é defendido
no terceiro valor que é mostrado no manifesto ágil.
    O último valor que é citado explica que o projeto deve sim ter um plano, porém as vezes deve mudar o plano, por conta da mudança de mercado ou questão do próprio cliente.
